# Story 卡片格式規範

## 檔名

`S{NNN}-{簡短標題}.md`

- `{NNN}` — 三位數流水號（S001, S002, ...）
- `{簡短標題}` — 中文或英文，用 `-` 連接

範例：`S013-工作設定模組化.md`

## 存放位置

由 `project-config.json` 的 `story_dir` 決定，預設 `文檔/stories`。

## 模板

```markdown
# {ID} {標題}

| 欄位 | 值 |
|------|------|
| 優先級 | 🔴 高 / 🟡 中 / 🟢 低 |
| 狀態 | 規劃中 / 進行中 / ✅ 已完成 / ⛔ 受阻 |
| 建立日 | {YYYY-MM-DD} |
| 完成日 | {YYYY-MM-DD} |
| 負責人 | {name} |
| 工作線 | {line} |

## 背景
{為什麼要做這件事}

## 前置確認
- [ ] {開工前需要確認的資訊}

## 子任務
- [ ] {具體步驟}

## 驗收條件
1. {可驗證的完成標準}

## 結案紀錄
{完成後補填：關鍵決策、產出物位置、技術備註}
```

## 狀態定義

| 狀態 | 意義 | 判斷規則 |
|------|------|---------|
| 規劃中 | 已建卡但未開工 | 子任務全部未勾 |
| 進行中 | 已開始執行 | 至少一個子任務已勾 |
| ✅ 已完成 | 全部做完並結案 | 子任務全勾 + 有結案紀錄 |
| ⛔ 受阻 | 被外部因素卡住 | 手動標記 |

## 活躍 story 判斷

session-start.sh 用以下規則判斷「活躍 story」：
- 掃描 `story_dir` 下所有 `S*.md` 檔案
- 讀取 metadata table 中的「狀態」欄位
- 不包含「已完成」的即為活躍

## 自動編號規則

`/story-new` skill 建立新 story 時：
1. 掃描 `story_dir` 下所有 `S*.md`
2. 取最大編號 + 1
3. 三位數補零（S001, S002, ..., S099, S100）

## 範例

見 SEO 工作區的 `文檔/stories/S002-西班牙檸檬汁桶裝上架.md`——完整的已結案 story。
